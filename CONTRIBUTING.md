## 한국어

### 개발 환경 설정

**요구 사항**: Python 3.10 이상 (`X | Y` union 문법 사용)

```bash
# 저장소 클론 후 프로젝트 루트에서 실행
python3 -m venv .venv
.venv/bin/pip install -r backend/requirements.txt

# 서버 시작
.venv/bin/python backend/app.py
# → http://127.0.0.1:19000 에서 확인
```

아트 에셋 재생성(선택):

```bash
cd assets-gen
pip install pillow          # 기본 아트
pip install rembg onnxruntime  # 고양이 생성 시 추가 필요
python3 gen_*.py
```

스모크 테스트(선택):

```bash
python3 scripts/smoke_test.py --base-url http://127.0.0.1:19000
```

### 프로젝트 구조

```
log8-office/
├── backend/          # Flask 백엔드
│   ├── app.py        # 진입점 (기본 포트 19000)
│   └── *_utils.py    # 기능별 유틸리티 모듈
├── frontend/         # Phaser 픽셀게임 프론트엔드
│   └── index.html    # 메인 UI (인라인 CSS/JS 단일 파일 — 게임 로직 전부 포함)
├── assets/           # 서빙되는 에셋 (WebP 스프라이트 등)
├── assets-gen/       # 픽셀킷 툴킷 — 에셋 생성 스크립트
├── scripts/          # 유틸리티 스크립트 (smoke_test.py 등)
└── docs/             # 추가 문서
```

---

### 코딩 컨벤션

#### 백엔드 (`backend/`)

- 진입점은 `app.py` 한 파일로 유지.
- 기능 단위 로직은 `*_utils.py` 모듈로 분리 (`agent_utils.py`, `auth_utils.py` 등).
- Python 3.10+ 기능(`X | Y` union, match/case 등) 사용 가능.
- Flask 블루프린트 없이 단순 라우팅 유지 (소규모 프로젝트).

#### 프론트엔드 (`frontend/`)

- `index.html`은 **단일 파일 원칙** — 인라인 CSS/JS로 유지 (업스트림 스타일). Phaser 씬·게임 로직 전부 여기 인라인.
- 새로운 별도 JS 파일 추가보다 기존 구조를 확장하는 방향 권장.
- Phaser 라이브러리는 `frontend/vendor/`에 고정 (CDN 의존 없음).

### 아트 에셋 — 반드시 읽어주세요

> **이 프로젝트는 외부 아트팩에서 가져온 비-CC0 아트를 절대 포함하지 않습니다.**

모든 아트는 `assets-gen/` 픽셀킷 스크립트로 생성된 CC0 자산입니다. 이는 의도적 원칙입니다:

- 업스트림(Star-Office-UI)의 비상업 라이선스 에셋과 LimeZu 스프라이트를 제거하고 완전 자체 제작으로 교체했습니다.
- 상업적 이용 포함 누구나 자유롭게 사용할 수 있도록 유지하기 위함입니다.

**아트 기여 규칙**:

1. 아트는 반드시 `assets-gen/gen_*.py` 스크립트를 통해 생성 또는 수정.
2. 외부 아트팩·스프라이트시트·아이콘팩에서 복사한 에셋은 PR에서 거부됩니다.
3. 새 에셋 스크립트를 추가하는 경우 `assets-gen/`에 넣고 라이선스를 CC0으로 명시.

### i18n 국제화

프론트엔드 `index.html` 내 `I18N` 객체에 번역이 집중 관리됩니다.

- 지원 언어: `ko` (기본) / `en` / `ja` / `zh`
- 신규 문자열 추가 시 **4개 언어를 동시에 추가**해야 합니다 (parity 원칙).
- 번역이 불확실한 경우 이슈에 표시하면 검토 후 반영합니다.

### 커밋 메시지 스타일

Conventional Commits 형식을 따릅니다. 한국어 커밋 메시지도 환영합니다.

```
feat: 에이전트 zone 전환 애니메이션 추가
fix: /health 엔드포인트 빈 state.json 처리
docs: 에이전트 연동 가이드 업데이트
refactor: auth_utils.py 분리
art: 고양이 달리기 스프라이트 개선
chore: 의존성 버전 업데이트
```

타입: `feat` / `fix` / `docs` / `refactor` / `art` / `chore` / `test`

---

### PR 프로세스

1. 저장소를 포크합니다.
2. 기능 브랜치를 생성합니다 (`feat/my-feature` 또는 `fix/issue-number`).
3. 변경 사항을 커밋합니다.
4. 스모크 테스트가 통과하는지 확인합니다:
   ```bash
   python3 scripts/smoke_test.py --base-url http://127.0.0.1:19000
   ```
5. `main` 브랜치 대상으로 PR을 열고 PR 템플릿을 채웁니다.
6. 아트를 추가/수정한 경우 PR에서 CC0 + assets-gen 생성 여부를 체크합니다.

### Coding Conventions

#### Backend (`backend/`)

- Keep the entry point in a single `app.py`.
- Split feature logic into `*_utils.py` modules (`agent_utils.py`, `auth_utils.py`, etc.).
- Python 3.10+ features (`X | Y` union, `match`/`case`, etc.) are fair game.
- Keep routing simple — no Flask blueprints (small project).

#### Frontend (`frontend/`)

- `index.html` follows the **single-file principle** — inline CSS and JS (matches upstream style). All Phaser scene/game logic lives inline here.
- Prefer extending the existing structure over adding new JS files.
- Phaser is pinned in `frontend/vendor/` — no CDN dependency.

---

### Art Assets — Please Read

> **This project ships zero art copied from external packs with non-CC0 licenses.**

All art is CC0, generated by the `assets-gen/` pixelkit scripts. This is a deliberate hard rule:

- The upstream Star-Office-UI included non-commercial-licensed assets and LimeZu sprites; log8-office replaced them entirely with self-generated CC0 art.
- The goal is to keep the project freely usable for everyone, including commercial use.

**Art contribution rules**:

1. All art must be created or modified via `assets-gen/gen_*.py` scripts.
2. Assets copied from external art packs, spritesheets, or icon packs will be rejected in PR review.
3. If you add a new generation script, place it in `assets-gen/` and mark the output as CC0.

---

### i18n Internationalization

Translations are centralized in the `I18N` object inside `frontend/index.html`.

- Supported locales: `ko` (default) / `en` / `ja` / `zh`
- When adding a new string, **add all four language entries at once** (parity rule).
- If you are unsure about a translation, flag it in the PR and a reviewer will help.

### PR Process

1. Fork the repository.
2. Create a feature branch (`feat/my-feature` or `fix/issue-number`).
3. Commit your changes.
4. Verify the smoke test passes:
   ```bash
   python3 scripts/smoke_test.py --base-url http://127.0.0.1:19000
   ```
5. Open a PR targeting `main` and fill out the PR template.
6. If you added or modified art, check the CC0 + assets-gen checkbox in the PR template.
