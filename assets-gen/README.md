# assets-gen — make your own office art (CC0)

Every pixel asset shipped in log8-office is **generated by the scripts in this
folder**, not drawn by hand or pulled from a third-party pack. That's why the
whole project is CC0 / commercial-friendly: the art is original and reproducible.

This doubles as the project's **"make your own assets" skill** — change the
palette or a generator, re-run it, and the office is re-skinned.

## Requirements

```bash
pip install pillow      # the only dependency
```

## Generate everything — one command

```bash
cd assets-gen
python3 build.py             # regenerate every asset into out/
python3 build.py --install   # …and copy it into the app (frontend/, assets/)
```

`build.py` runs all the generators in the right order and verifies every file's
exact pixel dimensions (animation frame counts derive from `image size ÷ frame
size`, so dimensions must be exact). Outputs land in `assets-gen/out/`
(git-ignored). The individual scripts still work if you want to run just one:

```bash
python3 gen_office_bg.py   # the fine 320x180 office background (canonical)
python3 gen_furniture.py   # sofa, desk, coffee machine, server, plants, posters, flowers, memo
python3 gen_decor.py       # extra props: clock, bookshelf, floor lamp, rug, whiteboard, water cooler
python3 gen_scene.py       # guests, cats, error-bug, button skins, room-reference
python3 gen_star.py        # the agent mascot: idle / working / sync animations
python3 gen_brand.py       # app icon (+ .ico) and README cover images
```

`gen_cats.py` is **not** part of `build.py` by default — it needs `rembg` +
`onnxruntime` and the source avatars. Pass `--cats` to include it:
`python3 build.py --cats`.

`--install` is safe: it never overwrites the named cat sprites
(`cat_*.webp` / `cat_*_role.png`, which only `gen_cats.py` produces), and the
canonical `office_bg` always comes from `gen_office_bg.py` (`build.py` runs it
last so its fine background wins over the coarse one `gen_scene.py` emits).

## Re-skin the whole office — themes

A **theme** overrides the palette so one config re-colours every sprite, prop and
piece of furniture at once. Themes live in `themes/` as JSON:

```json
{ "name": "midnight", "description": "...",
  "palette": { "wall": "#23263f", "teal": "#2fe6e0", "wood": "#4a5a72" } }
```

Only palette keys that already exist in `pixelkit` are honoured (unknown keys are
ignored). Four presets ship:

| theme       | look                                            |
|-------------|-------------------------------------------------|
| `default`   | the shipped cozy palette (template to copy)     |
| `midnight`  | dark cool — deep indigo walls, cyan/teal accents|
| `sakura`    | soft pastel pinks & creams                      |
| `forest`    | warm greens, rich woods, earthy                 |

Build with a theme:

```bash
python3 build.py --theme midnight --install
python3 build.py --theme themes/my-own.json   # or a path to your own
```

Under the hood `build.py` sets the `LOG8_THEME` env var; `pixelkit` auto-applies
it at import. You can also do it in code:

```python
import pixelkit
pixelkit.apply_theme("themes/sakura.json")   # or a dict, or pixelkit.load_theme(path)
```

To make your own theme, copy `themes/default.json`, edit the hex values, and
build with `--theme path/to/yours.json`.

## How it works — `pixelkit.py`

- **`PALETTE`** — the cozy-office colour set (brand blue `#7c83ff`, teal `#42d6a4`,
  warm woods, etc.). Edit this first to recolour the whole office — or drop in a
  theme (see above) via `apply_theme` / `load_theme` / the `LOG8_THEME` env var.
- **`Canvas`** — a small *logical-resolution* image (1 cell = 1 art pixel) with
  pixel-art helpers. Author small, then `.scaled(factor)` upscales with NEAREST so
  pixels stay crisp.
- **`Sheet(cols, rows, cell_w, cell_h)`** — packs frames into a spritesheet grid
  and `.save(path, target_w, target_h)` exports at the exact target size.
- **`outline_alpha`**, **`shade`**, **`soft_shadow`**, **`ease`**, **`wobble`** —
  the small primitives the generators lean on.

## Extra decor props (`gen_decor.py`)

Six new cozy, theme-aware props (colours come from `PALETTE`, so themes recolour
them too):

| file                   | size     | wired into the scene? |
|------------------------|----------|-----------------------|
| `whiteboard-v1.webp`   | 200×144  | yes — upper-center wall |
| `bookshelf-v1.webp`    | 160×200  | yes — far-right floor |
| `water-cooler-v1.webp` | 96×176   | yes — bottom-right corner |
| `wall-clock-v1.webp`   | 96×96    | drawer-swappable      |
| `floor-lamp-v1.webp`   | 96×224   | drawer-swappable      |
| `small-rug-v1.webp`    | 192×96   | drawer-swappable      |

The three wired props are added in `frontend/index.html` following the same
`load.image` + `add.image(x, y, key)` pattern as the existing furniture. The
other three ship as sprites you can swap in (e.g. via the asset drawer) without
touching code.
